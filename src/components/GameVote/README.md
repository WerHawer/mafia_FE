# GameVote Component

Компонент для відображення та керування списком гравців, які виставлені на голосування.

## Структура

### Файли:

- **GameVote.tsx** - Головний компонент-контейнер (presentation)
- **useGameVote.ts** - Кастомний хук з бізнес-логікою
- **VotePanel.tsx** - Компонент панелі голосування
- **VoteListItem.tsx** - Компонент окремого елемента списку
- **GameVote.module.scss** - Стилі для всіх компонентів

## Розділення відповідальності

### GameVote.tsx (Presentation)
Відповідає тільки за:
- Відображення кнопки toggle
- Рендеринг overlay
- Передачу даних у VotePanel
- Анімацію появи/зникнення

### useGameVote.ts (Logic)
Відповідає за:
- Керування станом (isOpen)
- Логіку голосування
- Перевірку прав голосування
- Визначення хто вже проголосував
- API виклики через мутації

### VotePanel.tsx (Presentation)
Відповідає за:
- Відображення панелі з анімацією
- Рендеринг заголовка та кнопки закриття
- Рендеринг списку VoteListItem
- Передачу props у дочірні компоненти

### VoteListItem.tsx (Presentation)
Відповідає за:
- Відображення одного елемента списку
- Обробку кліків та клавіатурної навігації
- Візуальні стани (clickable, voted)
- Показ індикатора голосу

## Використання

```tsx
import { GameVote } from "@/components/GameVote";

// У батьківському компоненті
<GameVote />
```

## Переваги архітектури

1. **Separation of Concerns** - логіка відокремлена від відображення
2. **Reusability** - кожен компонент можна переісповикористати окремо
3. **Testability** - логіку легко тестувати незалежно від UI
4. **Maintainability** - зміни в логіці не впливають на UI і навпаки
5. **Performance** - VoteListItem обгорнутий у memo для оптимізації
